# OctoAcme Project Management Docs — README

Welcome to the OctoAcme Project Management Docs! This README provides a brief introduction to how OctoAcme manages projects and links to detailed process documents.

## Recent Updates

This documentation has been enhanced to include expanded role definitions and cross-functional collaboration guidance:
- **Four new roles added**: Change Manager, Subject Matter Expert (SME), Support Engineer, and UX Designer
- **Enhanced checklists**: Added team onboarding, cross-functional coordination, and role-specific escalation paths
- **Improved clarity**: Role boundaries, responsibilities, and interactions clearly documented across all process docs

## At a Glance: OctoAcme Project Management Processes

- **Initiation:** Define the business need, align stakeholders, set goals, and document in a Project One-pager.
- **Planning:** Break down work, estimate scope, identify dependencies, plan releases and sprints, and define the "Definition of Done."
- **Execution & Tracking:** Use daily standups, reviews, and project boards to ensure delivery, track progress, and handle blockers.
- **Risks & Communication:** Maintain a risk register and clear communication channels; regular stakeholder updates and escalation paths ensure alignment.
- **Release & Deployment:** Follow checklists for safe releases, automated and manual testing, and incident response plans.
- **Retrospective & Improvement:** Capture learnings after milestones/releases and document action items for continuous improvement.
- **Roles & Personas:** Defined roles and responsibilities for all contributors, including cross-functional team members.
